# Getting Started

This guide will help you set up the complete CodeMentor AI platform locally.

CodeMentor AI is a full-stack MERN SaaS platform with AI/ML integration, smart matching systems, and secure payments.
---

## 📋 Prerequisites

Before setting up CodeMentor AI, make sure the following tools, services, and libraries are available in your development environment.

---

### 🖥 Core Runtime & Backend

- Node.js (v18+ recommended)
- npm or yarn
- Express.js
- MongoDB (Local installation or MongoDB Atlas)
- Git

---

### 🎨 Frontend & UI Libraries

- Tailwind CSS (Utility-first styling)
- Flowbite (Component library built on Tailwind)
- Framer Motion (Animations & transitions)
- React Hot Toast (Notifications & alerts)
- Radix UI (Accessible UI primitives)

---

### ✍️ Content & Editor Tools

- Quill Editor (Rich text editor for blogs)
- MDX (Documentation support)
- HTML decode / sanitize libraries (XSS protection)

---

### 🔐 Security & Encryption

- bcrypt (Password hashing)
- JWT (Authentication tokens)
- crypto / encryption utilities
- dotenv (Environment variable security)
- CORS middleware
- Helmet (Security headers)

---

### 💳 Payment & Monetization

- Razorpay account (for payments)
- Razorpay Webhooks configuration
- Subscription handling logic

---

### 🔐 Authentication

- Google OAuth credentials
- OAuth redirect configuration

---

### 🤖 AI & Machine Learning

- OpenAI API key (if AI features enabled)
- Cosine Similarity implementation (custom ML logic)
- Vector-based skill matching system

---

### 📦 Other Supporting Libraries

- Axios (API requests)
- Mongoose (MongoDB ODM)
- Multer (File uploads)
- Cloud storage integration (optional)
- Environment variable configuration (.env support)

---

## 1️⃣ Clone the Repository

```bash
git clone https://github.com/kartikrathore7773/CodeMentor AI.git
cd CodeMentor AI
```

---

## 2️⃣ Install Dependencies

### Install frontend dependencies

```bash
cd CodeMentor AI-frontend
npm install
```

### Install backend dependencies

```bash
cd ../CodeMentor AI-backend
npm install
```

---

## 3️⃣ Environment Variables Setup

Create a `.env` file inside the `server/` folder:

```env
PORT=3000
MONGO_URI=your_mongodb_connection_string
JWT_SECRET=your_secret_key

GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret

STRIPE_SECRET_KEY=your_stripe_key
RAZORPAY_KEY_ID=your_razorpay_key
RAZORPAY_SECRET=your_razorpay_secret

OPENROUTER_API_KEY=your_ai_key
```

Create a `.env.local` file inside `CodeMentor AI-frontend/`:

```env
NEXT_PUBLIC_API_URL=http://localhost:3000/api
NEXT_PUBLIC_STRIPE_PUBLIC_KEY=your_stripe_public_key
```

---

## 4️⃣ Run the Backend

```bash
cd server
npm run dev
```

Backend will start at:

```
http://localhost:3000
```

---

## 5️⃣ Run the Frontend

```bash
cd CodeMentor AI-frontend
npm run dev
```

Frontend will start at:

```
http://localhost:3001
```

---

## 🤖 AI Engine Configuration

The AI engine supports:

- Blog draft assistance
- Cosine similarity skill matching
- Job recommendation logic
- Hackathon evaluation scoring

Ensure:

- AI API key is valid
- Server has access to AI services
- MongoDB contains user skill vectors (if cosine matching enabled)

---

## 💳 Payment Configuration

CodeMentor AI supports:

- One-time payments
- Subscription memberships
- Hackathon entry payments

To enable payments:

1. Configure Stripe or Razorpay keys
2. Set webhook endpoint in payment dashboard
3. Enable production keys before deployment

---

## 🔐 Authentication Setup

- Configure Google OAuth in Google Cloud Console
- Add authorized redirect URIs:

  ```
  http://localhost:3000/api/auth/callback/google
  ```

- Add credentials in `.env`

---

## 🧠 Cosine Similarity Matching (Optional Setup)

If using vector-based matching:

- Store user skills as vectors in MongoDB
- Precompute job vectors
- Run similarity scoring inside `ai-engine/`

---

## 🚀 Production Build

### Frontend

```bash
npm run build
```

### Backend

```bash
cd CodeMentor AI-backend
npm run dev
```

---

## 🛠 Common Issues

### MongoDB connection error

- Check URI
- Ensure IP whitelisted (Atlas)

### OAuth redirect mismatch

- Verify redirect URL in Google Console

### Payment not triggering webhook

- Ensure webhook endpoint configured correctly
